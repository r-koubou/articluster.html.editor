# タスクログ 2026-05-06

## 作業内容：MIDIチャンネル情報の仕様変更対応

### 変更ファイル

- `apps/app.js`

### 変更内容

#### 方針確認

- `Channel`（大文字、独立フィールド）と `channel`（小文字、ステータスバイト埋め込み）を整理し、`channel`（小文字）を廃止
- `Status` バイトはメッセージ種別のみ（下位ニブルのチャンネル埋め込みなし）とする
- 既存 YAML の Status に埋め込まれたチャンネル情報の消失は許容

#### コード変更

| 箇所 | 変更内容 |
|---|---|
| `encodeStatus(type, channel)` | `channel` 引数を削除、ベース値のみ返す |
| `decodeStatus(byte)` | 下位ニブル無視、`type` のみ返す |
| `newMidiMessage()` | `channel: 0` → `Channel: -1` |
| UI（MIDI行表示順） | `Status → Channel → Data1 → Data2` → `Channel → Status → Data1 → Data2` |
| UI（Channel入力） | 旧 "Ch"（0〜15）を "Channel"（-1〜15、placeholder: -1）に変更 |
| YAML エクスポート | `Channel !== -1` の場合のみ `Channel:` を出力 |
| YAML インポート | `Channel` フィールドがあれば読込、なければ `-1` をセット |

### 動作確認結果

- 新規 MIDI Message 追加時：Channel 欄が -1 で表示される
- Channel 指定あり YAML インポート：Channel 値が正しく表示される
- Channel 指定なし YAML インポート：Channel 欄が -1 になる
- エクスポート：Channel が -1 の場合は YAML に出力されない、0〜15 の場合は出力される
- MIDI 行の表示順：Channel → Status → Data1 → Data2

すべて正常動作を確認。
